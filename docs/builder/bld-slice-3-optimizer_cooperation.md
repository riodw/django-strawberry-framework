### Notes for Worker 1 (spec reconciliation)

**STRUCTURAL DRIFT — sub-check #1's verbatim assertion is unsatisfiable in this card's scope; status held at `revision-needed`.**

- **The fact.** In `0.0.9` the flat optimizer walker is connection-unaware: given a `DjangoConnectionField`'s `info`, its root children are `edges` / `pageInfo` (not node fields), and `_walk_selections` looks those names up in the NODE MODEL's `field_map`, finds nothing, and descends no further. So `apply_connection_optimization` runs, but the `OptimizationPlan` it derives is **EMPTY for every connection field** — no `select_related`, no `prefetch_related`, AND no `only()` scalar projection — even for a direct forward FK (`category`) under `edges { node }`.
