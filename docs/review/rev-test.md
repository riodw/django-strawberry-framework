- **New helpers considered.** A sibling consumer-helper was considered and rejected at folder scale: (1) a `safe_unwrap_connection_method(connection, method_name) -> bool` symmetric to the wrap-time helper, returning the saved original method to the consumer — rejected because the consumer's `try/finally` save-and-restore idiom (per `_wrap.py:81-116` docstring example) is a documented Django pattern with no `_DatabaseFailure`-style sibling risk on the unwrap path; the unwrap-time backstop already lives in `_django_patches.py` as the package-level patch; (2) a `SafeConnectionMethodWrap(connection, method_name, wrapper)` context manager wrapping the save-and-restore boilerplate — rejected because the consumer's `setUp` / `tearDown` pair is the canonical Django `TestCase` shape and a context manager would re-implement the test-case lifecycle at a third layer.
